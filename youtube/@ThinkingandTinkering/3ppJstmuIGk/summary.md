## Overview
Robert Murray-Smith shares his philosophy on learning 3D CAD software, arguing that learning should be enjoyable and progressive rather than forced. He advocates starting with simpler tools like TinkerCAD to build spatial intuition before moving to parametric programs like Onshape, and emphasises understanding global concepts over program-specific details.

## Key Topics
- The importance of making learning fun and building on incremental successes
- How physical play (Play-Doh, Lego, Meccano) develops the spatial reasoning needed for 3D CAD
- The difference between navigating 3D space in the real world vs. constructing objects in a CAD workspace
- TinkerCAD as a beginner-friendly stepping stone using pre-made primitives
- Onshape as a parametric CAD program where primitives are defined by parameters (length, width, height)
- The value of a "global" understanding of CAD concepts to avoid being tied to any single program

## Materials and Chemicals Mentioned
None mentioned.

## Techniques and Methods
- Using primitive shapes (spheres, cubes) in TinkerCAD to learn workspace navigation
- Parametric modelling in Onshape: defining primitives by numerical parameters, then adding features
- Progressive skill-building: starting with physical/tactile construction toys before moving to digital tools
